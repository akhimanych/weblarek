# Проектная работа "Веб-ларек"

Стек:

- **HTML** — базовая разметка страниц.
- **SCSS** — стилизация компонентов интерфейса.
- **TypeScript (TS)** — типизация кода и базовые интерфейсы.
- **Webpack** — сборка и организация модулей приложения.

Структура проекта:

- `src/` — исходные файлы проекта
- `src/components/` — папка с JS компонентами
- `src/components/base/` — папка с базовым кодом

Важные файлы:

- `src/pages/index.html` — HTML-файл главной страницы
- `src/types/index.ts` — файл с типами
- `src/index.ts` — точка входа приложения
- `src/scss/styles.scss` — корневой файл стилей
- `src/utils/constants.ts` — файл с константами
- `src/utils/utils.ts` — файл с утилитами

## Данные и типы данных

### Типы данных приложения

- **Продукт** (`interface IProduct`)  
  Определяет формат данных для товаров в системе. Применяется для отображения информации о продуктах в каталоге и корзине.

  Данные продукта:
  `id: string;` // Уникальный идентификатор товара
  `description: string;` // Описание товара
  `image: string;` // Картинка товара
  `title: string;` // Название товара
  `category: string;` // Категория товара
  `price: number | null;` // Цена товара; null — если товар бесценен

---

- **Заказ** (`interface IOrder`)
  Хранит данные о заказе, который находится в процессе оформления. Применяется для формирования и передачи заказа на сервер.

  Данные заказа:
  `payment: string` // способ оплаты заказа
  `email: string;` // Email покупателя
  `phone: string;` // Телефон покупателя
  `address: string;` // Адрес доставки
  `total: number;` // Общая сумма заказа
  `items: string[];` // Массив идентификаторов товаров в заказе

---

- **Информация о заказе** (`interface IOrderInfo`)  
  Используется для отображения подтверждения заказа после оформления.

  Подтверждение заказа:
  `id: string;` // Уникальный идентификатор заказа
  `total: number;` // Итоговая сумма заказа

---

- **Ответ с сервера** (`interface IGetProductListResponce`)  
  Формат данных, возвращаемых сервером при запросе списка товаров. Применяется для обработки результата API при загрузке каталога.

  Результат запроса каталога:
  `total: number;` // Общее количество товаров в каталоге
  `items: TProductList;` // Массив товаров каталога

---

- **Способы оплаты** (`TOrderPayment`)  
  Содержит перечень доступных методов оплаты. Применяется для проверки корректности и обработки выбора пользователя при оплате заказа.

  Значения:
  `type TOrderPayment = "online" | "upon_receipt";`

---

- **Список продукта** (`TProductList`)  
  Задает формат данных для массива товаров. Применяется для задания типа списков товаров в разных модулях приложения.

  `type TProductList = IProduct[];`

Все интерфейсы хранятся в `src/types/index.ts` и используются для типизации моделей, компонентов и View.

## Архитектура приложения

Приложение реализовано по паттерну MVP (Model–View–Presenter)

### MVP: разделение слоев

- **Model (слой данных):**
  Работа с данными в проекте реализована с помощью следующих классов — каждый отвечает за свою зону ответственности и работает с определённым типом данных. Валидируют данные и эмитят события об изменениях через EventEmitter.

---

#### Класс ModelProductList

Отвечает за хранение списка товаров (каталога) и предоставляет методы для доступа к данным.

**_Конструктор_**
`constructor(events: IEvents)`
• `events` — экземпляр брокера событий для генерации и подписки на события.

**_Поля_**
• `items: IProduct[]` — массив товаров каталога.
• `events: IEvents` — объект для рассылки событий об изменениях.

**_Методы_**
• `getItems(): any[]` - Возвращает список всех товаров каталога.
• `setItems(items: any[]): void` - Обновляет каталог товаров и генерирует событие `items:changed`.
• `getItemById(id: string): IProduct | undefined` - Возвращает товар по id или undefined, если не найден.

---

#### Класс ModelOrder

Хранит данные заказа и обеспечивает управление и валидацию этих данных.

**_Конструктор_**
`constructor(events: IEvents)`

**_Поля_**
• `_order: IModelOrder` — объект с данными текущего заказа.
• `events: EventEmitter ` — система событий.

**_Методы_**
• `setPayment(method: TOrderPayment): void` - Устанавливает способ оплаты, вызывает событие `order.payment:changed`.
• `setAddress(address: string): void` - Устанавливает адрес доставки, вызывает событие `order.address:changed`.
• `setEmail(email: string): void` - Устанавливает email, вызывает событие `order.email:changed`.
• `setPhone(phone: string): void` - Устанавливает телефон, вызывает событие `order.phone:changed`.
• `setOrder(data: Partial<IModelOrder>): void` - Обновляет данные заказа, вызывает событие `order:changed`.
• `getOrder(): IModelOrder` - Возвращает текущие данные заказа.
• `clearOrder(): void` - Сбросить все поля заказа к начальному состоянию.
• `validateOrder(): boolean` - Проверяет заполненность всех обязательных полей.Возвращает объект с ошибками для каждого неверного поля

---

#### Класс ModelCart

Управляет корзиной пользователя: добавляет и удаляет товары, рассчитывает стоимость, уведомляет представление о изменениях.

**_Конструктор_**
`constructor(events: IEvents)`

**_Поля_**
• `items: IProduct[]` — массив товаров в корзине.
• `events: IEvents` — объект событий.

**_Методы_**
• `addCart(product: IProduct): void` - Помещает товар в корзину и уведомляет об этом событием.
• `deleteCart(id: string): void` - Убирает товар из корзины по ID и отправляет событие `cart:changed`.
• `getItems(): IProduct[]` - Возвращает список всех товаров из корзины.
• `getTotalCount(): number` - Считает количество товаров в корзине.
• `getTotal(): number` - Подсчитывает общую сумму корзины.
• `hasItem(id: string): boolean` - Проверяет, есть ли товар в корзине по ID.
• `clearCart(): void` - полностью очищает корзину и отправляет событие `cart:changed`.

---

- **View (слой представления):**

  - Компоненты отображения (`ViewPage`, `ViewModal`, `ViewCartModal`, `ViewCartProductItem`, `ViewOrderContactModal`, `ViewOrderModal`, `ViewProductItem`, `ViewProductModal`, `ViewSuccessModa`l)
  - Каждый класс рендерит данные внутри DOM-элемента и реагирует на ввод пользователя
  - Отправка событий при действиях пользователя (клик, ввод, submit).

- **Presenter (слой связи/логики):**
  - Файл `index.ts`:
    - Создает экземпляры моделей и представлений
    - Подписывает обработчики событий
    - Реализует бизнес-логику обработки действий пользователя и синхронизации данных
  - Использует EventEmitter для централизованной связи между View и Model.

## Базовый код

- **Api** — отправляет запросы к серверу. В конструктор передаётся адрес сервера и (по желанию) заголовки.
  • `get`  — запрашивает данные с сервера по адресу и возвращает ответ
  • `post`  — отправляет данные на сервер

- **EventEmitter** — брокер событий, реализует подписку, вызов и снятие событий через интерфейс `IEvents` •
  • `on` - подписаться на событие
  • `emit` - отправить событие
  • `trigger` - возвращает функцию, которая при вызове отправит событие

## Описание компонентов

### View

- **ViewPage**
  Класс наследует базовый  Component  и отвечает за отрисовку главного экрана приложения. В нём формируется каталог товаров, отображается и обновляется счётчик корзины, а также обрабатываются пользовательские взаимодействия.

Основные элементы:
•  `.gallery`  — контейнер карточек товаров ( productContainer ).
•  `.header__basket`  — кнопка корзины в шапке ( cartButtonElement ).
•  `.header__basket-counter`  — счётчик количества товаров ( counterElement ).

Ключевые методы:
• Свойство‑сеттер  catalog  принимает массив HTML‑элементов карточек и отрисовывает их в  `.gallery` .
• Свойство‑сеттер  counter  обновляет значение счётчика в корзине.
• По клику на кнопку корзины генерируется событие  `cart:open` , которое открывает модальное окно корзины.

- **ViewModal**
  Класс реализует базовую логику модального окна и предназначен для открытия/закрытия диалогов и установки контента внутри них.

Основные элементы:
•  `.modal__content`  — контейнер для содержимого модального окна ( _modalContent ).
•  `.modal__close`  — кнопка закрытия окна ( _closeButton ).

Ключевые метод_similarity:
• Свойство‑сеттер  content  устанавливает любой HTML‑элемент в качестве содержимого модального окна.
• Метод  open(element?: HTMLElement)  открывает окно, при необходимости подставляя указанный контент.
• Метод  close()  закрывает окно и очищает содержимое.

Пользовательские события:
• Клик по кнопке  `.modal__close`  и по затемнённой области вокруг модального окна вызывает  close() .

- **ViewCartModal**
  Класс отвечает за отображение корзины товаров, общей стоимости и управления состоянием кнопки оформления заказа.

Основные элементы:
•  `.basket__button`  — кнопка оформления заказа ( _orderButton ).
•  `.basket__price`  — элемент общей стоимости ( _totalPrice ).
•  `.basket__list`  — контейнер списка товаров ( _listElement ).
•  `.basket__item-delete`  — кнопки удаления товаров (по  data-id ).

Ключевые методы:
• Свойство‑сеттер  `products`  принимает массив HTML‑элементов карточек товаров и отображает их в списке, а также переключает состояние кнопки оформления.
• Свойство‑сеттер  `total`  устанавливает итоговую сумму заказа.

Пользовательские события:
• Клик по кнопке оформления заказа генерирует событие  cart:order .

- **ViewCartProductItem**
  Класс отображает отдельный товар в корзине, его цену, порядковый номер и обеспечивает возможность удаления из корзины.

Основные элементы:
•  `.basket__item-index`  — индекс товара ( _indexElement ).
•  `.card__title`  — название товара ( _titleElement ).
•  `.card__price`  — цена ( _priceElement ).
•  `.basket__item-delete`  — кнопка удаления ( _buttonElement ).

Ключевые методы:
• Свойство‑сеттер  `product`  устанавливает объект товара (название и цену).
• Свойство‑сеттер  `counter`  задаёт порядковый номер товара в списке.

Пользовательские события:
• Клик по кнопке удаления генерирует событие  `cart:delete-item`  с идентификатором товара.

- **ViewOrderContactModal**
  Класс отвечает за ввод контактных данных заказчика и управление валидацией формы.

Основные элементы:
•  `name="email"`  — поле ввода email ( _inputEmail ).
•  `name="phone`"  — поле ввода телефона ( _inputPhone ).
•  `buttontype="submit`"  — кнопка отправки формы ( _submitButton ).

Ключевые методы:
• Метод  `isLoading(state: boolean)`  блокирует/разблокирует кнопку отправки и меняет её текст.
• Приватные методы  `isFormValid()`  и  `updateSubmitButtonState()`  следят за тем, что оба поля заполнены.

Пользовательские события:
• Изменение значений в  email  и  phone  генерирует событие  `inputUser:changed` .
• При валидной форме клик по кнопке отправки генерирует событие  `finish:click` .

- **ViewOrderModal**
  Класс  управляется отдельным шаблоном и отвечает за выбор способа оплаты и ввод адреса доставки.

Основные элементы:
•  `.order`  — форма заказа ( _order ).
•  `name="card"`  — кнопка онлайн‑оплаты ( _cardButton ).
•  `name="cash"`  — кнопка оплаты при получении ( _cashButton ).
•  `name="address"`  — поле адреса ( _inputAddress ).
•  `buttontype="submit"`  — кнопка перехода к следующему шагу ( _submitButton ).

Ключевые методы:
• Метод  `setPaymentMethod(method: 'card' | 'cash')`  меняет выбранный способ оплаты и визуально подсвечивает соответствующую кнопку.
• Приватные методы  `isFormValid()`  и  `updateSubmitButtonState()`  проверяют заполненность адреса и наличие выбранного способа оплаты.

Пользовательские события:
• Клики по кнопкам  card  и  cash  генерируют  `inputUser:changed`  с указанием способа оплаты.
• Изменение адреса также генерирует  `inputUser:changed` .
• При валидной форме клик по кнопке отправки генерирует событие  `order:proceed` .

- **ViewProductItem**
  Класс отвечает за отображение карточки товара в каталоге: название, категория, цена и изображение.

Основные элементы:
•  `.card__title`  — название товара ( titleElement ).
•  `.card__image`  — изображение ( imageElement ).
•  `.card__category`  — категория ( categoryElement ).
•  `.card__price`  — цена ( priceElement ).

Ключевые особенности:
• Свойство‑сеттер  product  устанавливает объект товара, применяет CSS‑класс категории (например,  `card__category_soft` ,  `card__category_hard` ,  `card__category_other` ,  `card__category_additional` ,  `card__category_button` ) и форматирует отображение цены.
• Для товаров без цены вместо цифры отображается текст «Бесценно».

Пользовательские события:
• Клик по карточке генерирует событие  `item:click`  с идентификатором товара.

- **ViewProductModal**
  Класс  отображает детальную информацию о товаре и управляют добавлением/удалением из корзины.

Основные элементы:
•  `.card__title` ,  `.card__text` ,  `.card__image` , `.card__category` ,  `.card__price`  — детали товара.
•  `.card__button`  — кнопка действия (купить/убрать из корзины,  _buttonElement ).

Ключевые поля:
•  `_product`  — данные товара.
•  `_selectedCart`  — флаг, отмечает, находится ли товар в корзине.

Ключевые методы:
• Свойство‑сеттер  `product`  устанавливает все данные товара, применяет класс категории и актуализирует состояние кнопки.
• Свойство‑сеттер  `selectedCart`  переключает текст кнопки между «Купить» и «Убрать из корзины», а также активирует/дизейбле её.

Пользовательские события:
• Клик по кнопке генерирует  `cart:delete-item`  (если товар в корзине) или  `cart:add-item`  (если его нет).
Особенности:
• Для товаров без цены кнопка блокируется, а текст заменяется на «Недоступно».

- **ViewSuccessModal**
  Класс отображает результат успешного оформления заказа и сумму списания.

Основные элементы:
•  `.order-success__close`  — кнопка закрытия ( _buttonClose ).
•  `.order-success__description`  — текст с информацией о списанной сумме ( _totalPrice ).

Ключевые методы:
• Свойство‑сеттер  totalPrice  устанавливает текст вида «Списано {сумма} синапсов».

Пользовательские события:
• Клик по кнопке закрытия генерирует событие  `closes:click` .

## Взаимодействие компонентов

- Все события (изменения данных, действия пользователя) реализованы через EventEmitter
- Ключевые события:
  - Изменение данных:
    •  `items:changed`  — обновился список товаров в каталоге
    •  `cart:changed`  — изменилось содержимое корзины
    •  `order.payment:changed`  — сменился способ оплаты
    •  `order.address:changed`  — поменялся адрес доставки
    •  `order.email:changed`  — обновился email покупателя
    •  `order.phone:changed`  — изменился номер телефона
    •  `order:changed`  — изменились данные заказа
  - Действия пользователя:
    •  `item:click`  — клик по товару для просмотра деталей
    •  `cart:add-item`  — добавить товар в корзину
    •  `cart:delete-item`  — убрать товар из корзины
    •  `cart:open`  — открыть корзину
    •  `cart:order`  — начать оформление заказа
    •  `order:proceed`  — перейти к контактным данным
    •  `finish:click`  — отправить заказ
    •  `closes:click`  — закрыть модальное окно
    •  inputUser:changed  — изменение полей ввода
  - View реагирует на действия пользователя и выбрасывает событие
  - Presenter обрабатывает событие, вызывает логику модели
  - Model обновляет состояние, сообщает об изменениях через события
  - View получает новые данные и обновляет отображение.
